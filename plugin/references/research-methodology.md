# Research Methodology

Shared approach for web and codebase research. Use this to gather facts quickly, verify important claims, and return an answer the caller can act on.

## Research Approach

1. Define the research objective before searching.
2. Decide whether the answer lives on the web, in project files, or both.
3. Search broadly enough to avoid tunnel vision, then narrow quickly.
4. Verify important claims across more than one source when stakes are meaningful.
5. Separate evidence from inference in the final output.

## Query Optimization

- use exact phrases in quotes when the wording matters
- include version, timeframe, or platform when those affect the answer
- run a few targeted variations instead of one vague query
- refine based on what the first results reveal

## Domain Filtering

- prefer official and primary sources
- scope to specific domains when authority matters
- use academic or standards sources for research topics
- avoid low-signal sources unless they provide evidence unavailable elsewhere

## Deep Dive Techniques

- open only the most promising results
- follow citations or references when they improve confidence
- note contradictions explicitly instead of averaging them away
- capture unstable facts with source/date context

## Project File Research

- use fast file search to locate relevant code and docs
- read the smallest set of files that can answer the question
- cross-check local documentation against the actual code when possible
- prefer pattern discovery over broad codebase tours

## Output Format

Return:
- **Objective**: what was researched
- **Method**: key queries, files, or sources used
- **Findings**: facts, contradictions, and confidence level
- **Recommendations**: practical next steps based on the evidence
- **References**: URLs or file paths needed for follow-up
